# DailyPulse News 📰

A modern, category-based news reader built with **Flutter**, **GetX** (state management + routing), and the **NewsAPI** REST API. Browse top headlines by category, search articles in real time, bookmark stories for later, and switch between light/dark themes — all with a clean MVC-style codebase.

---

## ✨ Features

- Home screen with 7 news categories (Top Headlines, Technology, Sports, Entertainment, Business, Health, Science)
- Infinite-scroll pagination + pull-to-refresh on category lists
- Real-time (debounced) search across all NewsAPI articles
- Article details screen with Hero image transition, bookmarking, and "Read Full Article" (opens the original URL)
- Bookmarks screen with swipe-to-remove, persisted locally via `SharedPreferences`
- Settings screen: light/dark theme + adjustable font size, both persisted
- Shimmer loading skeletons, dedicated error states, and empty states everywhere data is fetched
- Graceful handling of no internet, timeouts, and API errors

## 🚀 Getting Started

### 1. Prerequisites

- [Flutter SDK](https://docs.flutter.dev/get-started/install) (latest stable channel)
- A code editor (VS Code or Android Studio recommended, with the Flutter/Dart plugins)
- A free [NewsAPI](https://newsapi.org/register) account

### 2. Get the code into a runnable Flutter project

This package contains the **`lib/`** folder and **`pubspec.yaml`** only — the platform folders (`android/`, `ios/`, `web/`, etc.) are intentionally not included, since they're auto-generated by the Flutter tool and shouldn't be hand-written. To turn this into a runnable project:

```bash
# 1. Create a fresh Flutter project shell
flutter create daily_pulse_news
cd daily_pulse_news

# 2. Replace the generated lib/ and pubspec.yaml with the ones from this download
#    (delete the generated lib/ and pubspec.yaml first, then copy these in)
rm -rf lib pubspec.yaml
cp -r /path/to/downloaded/news_app/lib .
cp /path/to/downloaded/news_app/pubspec.yaml .
cp /path/to/downloaded/news_app/analysis_options.yaml .   # optional, for linting
```

### 3. Install dependencies

```bash
flutter pub get
```

### 4. 🔑 Add your NewsAPI key

1. Sign up for a free key at **https://newsapi.org/register**.
2. Open `lib/app/utils/api_constants.dart`.
3. Replace the placeholder:

```dart
static const String apiKey = "YOUR_API_KEY"; // 👈 put your real key here
```

> **Note on the free NewsAPI plan:** the free tier only allows requests from `localhost` and has a daily rate limit, and the `/everything` (search) endpoint on the free tier only returns articles from the last month. This is a NewsAPI limitation, not an app bug — if search or headlines return a 426/429 error, see NewsAPI's docs for current free-tier rules.

### 5. Run the app

```bash
flutter run
```

## 🏗 MVC Architecture Explanation

This project follows an **MVC-flavored GetX structure**, adapted for Flutter:

- **Model** — `lib/app/data/models/article_model.dart` defines `Article`, `Source`, and `NewsResponse`: plain Dart classes that parse and hold NewsAPI JSON data. Models have no UI or business logic.
- **View** — every `*_view.dart` file under `lib/app/modules/*/views/`. Views are "dumb": they only read reactive state from a controller and render widgets. They never call the API or touch `SharedPreferences` directly.
- **Controller** — every `*_controller.dart` file under `lib/app/modules/*/controllers/`. Controllers extend `GetxController`, hold `.obs` reactive state (loading/error/data), and call into the **services** layer to fetch or persist data. This is the "glue" between Model and View.

On top of strict MVC, there's a small **service/provider layer** (`lib/app/data/providers` and `lib/app/data/services`) that keeps networking/storage code out of controllers entirely:

- `ApiProvider` — the only place that builds URLs, calls `http.get`, and turns network/API failures into a friendly `ApiException`.
- `NewsService` — domain-level wrapper that turns raw JSON into typed `NewsResponse`/`Article` objects for controllers to consume.
- `BookmarkService` / `ThemeService` — wrap `SharedPreferences` reads/writes for bookmarks and theme/font preferences.

This means a controller never sees raw JSON or `http` exceptions — only typed models or a clean error message.

## ⚙️ GetX Workflow Explanation

GetX is used for three things in this app: **state management**, **routing**, and **dependency injection**.

1. **Dependency injection (Bindings)**
   Each feature module has a `*_binding.dart` that registers its controller with `Get.lazyPut<T>()` right before its view is shown. App-wide singletons (`NewsService`, `BookmarkService`, `ThemeService`) are registered once in `InitialBinding`, attached to `GetMaterialApp.initialBinding` in `main.dart`, and looked up anywhere with `Get.find<T>()`.

2. **State management (`.obs` + `Obx`)**
   Controllers expose state as `RxType` fields (e.g. `final articles = <Article>[].obs;`, `final state = ViewState.loading.obs;`). Views wrap the parts of the UI that depend on that state in `Obx(() => ...)`, so only those widgets rebuild when the value changes — no `setState()` needed.

3. **Routing (`GetMaterialApp` + named routes)**
   All routes are declared once in `app_pages.dart` as a list of `GetPage`s, each pointing to a view + binding. Navigation anywhere in the app is a one-liner: `Get.toNamed(AppRoutes.article, arguments: article)`. Data is passed between screens via `Get.arguments` (see `CategoryController`/`ArticleController` reading `Get.arguments` in `onInit()`), so there's no need for constructors with required parameters on every view.

Typical request flow (e.g. opening a category):

```
HomeView → controller.openCategory(category)
  → Get.toNamed(AppRoutes.category, arguments: category)
    → CategoryBinding registers CategoryController
      → CategoryController.onInit() reads Get.arguments, calls NewsService
        → NewsService → ApiProvider → http.get(NewsAPI)
      → state.value updates (loading → loaded/error/empty)
    → CategoryView's Obx() rebuilds to show the new state
```

## ⚠️ Known Limitations

- NewsAPI's **free developer plan** restricts the `/everything` search endpoint to articles published in roughly the last month, and top-headlines/search calls are rate-limited. Upgrade to a paid plan for production use.
